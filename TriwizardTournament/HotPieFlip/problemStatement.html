In an alternate Universe, Queen Cersei has won the battle for Westeros. Even the white walkers have bent their Skeletal knee.

To celebrate such an amazing victory, a Grand Feast has been planned at King's Landing. The Queen has ordered the "Inn at the Crossroads" to make N number of pies for the occasion. "Hot-Pie", the famous bread maker, now oversees the functioning of "Inn at the Crossroads" and so the responsibility falls onto him.

Each Pie has two sides, the top side (with all the toppings) and the base. The pies are cooked in a single row over a hot surface. In haste, Hot-Pie has placed the pies randomly (i.e. Somewhere the Top is facing up, somewhere the Base is facing up)

In order to be properly cooked, all the pies need to be with Top side facing up. Unfortunately, Hot-Pie does not have a regular Pie flipper at disposal. All the utensils are in use by other Chefs, except for an oversized Pie Flipper that flips exactly K consecutive pies. That is, in that range of K pies, it changes every Top-side pie to a Back-side pie, and vice versa; it does not change the left-to-right order of those pies.

You cannot flip fewer than K pies at a time with the flipper, even at the ends of the row (since there are raised borders on both sides of the cooking surface). For example, you can flip the first K pies, but not the first K - 1 pies.

Since there is very little time left for the feast, all the pies need to be brought into Top-side up position as soon as possible. Hot-Pie, as you may know, is not really good with problem solving and hence has asked you for help.

Given the current state of the pies, calculate the minimum number of uses of the oversized pie flipper needed to leave all pies Top-side up, or state that there is no way to do it.

 

Input

The first line of the input gives the number of test cases, T. T test cases follow. Each consists of one line with a string S and an integer K. S represents the row of pies: each of its characters is either + (which represents a pie that is initially Top-side up) or - (which represents a pie that is initially Back-side up).

 

Output

For each test case output an integer representing the minimum number of times you will need to use the oversized pancake flipper to get all the pies Top-side up or print "IMPOSSIBLE" (without quotes) if there is no way to do it.

 

Constraints

  • 1 ≤ T ≤ 100
  • Every character in S is either + or -
  • 2 ≤ K ≤ length of S

 

Subtasks

  • Subtask 1 (30 points) : 2 ≤ length of S ≤ 10
  • Subtask 2 (70 points) : 2 ≤ length of S ≤ 1000

Example

Input:
3
---+-++- 3
+++++ 4
-+-+- 4

Output:
3
0
IMPOSSIBLE

 

Explanation

Test Case 1 :

We can get all the pies Top-side up by first flipping the leftmost 3 pies, getting to ++++-++-, then the rightmost 3, getting to ++++---+, and finally the 3 pies that remain Back-side up. There are other ways to do it with 3 flips or more, but none with fewer than 3 flips.

Test Case 2 :

All of the pies are already Top-side up, so there is no need to flip any of them.

Test Case 3 :

There is no way to make the second and third pies from the left have the same side up, because any flip flips them both. Therefore, there is no way to make all of the pies Top-side up.
